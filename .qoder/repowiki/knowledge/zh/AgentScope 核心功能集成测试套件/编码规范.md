- 所有异步测试类均继承自 `unittest.async_case.IsolatedAsyncioTestCase`，并统一实现 `asyncSetUp` 和 `asyncTearDown` 以管理测试生命周期。
- 在涉及外部 API 调用的模型测试中，普遍使用 `unittest.mock.patch` 装饰器拦截底层客户端（如 `openai.AsyncClient`），并注入预定义的 `MagicMock` 响应对象。
- 测试断言广泛采用 `AnyString()` 等自定义匹配器处理动态生成的 ID 或时间戳，确保对非确定性字段的灵活比对。
- 复杂场景测试（如 HITL 或多步工具调用）遵循“事件收集-状态比对”模式，通过遍历 `reply_stream` 收集事件列表并与预期的字典结构进行深度比对。
